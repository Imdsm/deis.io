Deploy Your Own Private PaaS

Before you get started, read about Deis core Concepts. Review our documentation for more detailed instructions. Please note that Deis requires a Chef Server. Don’t have one? Sign up for a free Hosted Chef account.

Want help? OpDemand provides installation assistance and commercial support.

01. Provision a Deis Controller

{% highlight bash %} $ git clone https://github.com/opdemand/deis.git # clone the repo $ cd deis && bundle install # install ruby deps $ berks install && berks upload # upload cookbooks $ contrib/provision-ec2-controller.sh # provision it $ sudo pip install deis # install the CLI $ deis register http://my.controller # register a user {% endhighlight %}

Provision a private Controller — the brains of a Deis application platform.

Use the Deis command-line client to interact with the controller.

02. Create & Scale a Formation

{% highlight bash %} $ deis formations:create dev --flavor=ec2-us-west-2 Creating formation... done, created dev Creating runtime layer... done in 1s Use `deis nodes:scale dev runtime=1` to scale a basic formation $ deis nodes:scale dev runtime=2 Scaling nodes... but first, coffee! ...done in 251s {% endhighlight %}

Create Formations of nodes, containers and proxies used to service applications.

Scale nodes automatically to add or remove capacity to a formation.

Add nodes to a formation manually where automated provisioning is not possible.
